# AI 销售顾问平台 - 系统设计规格

> 日期：2026-04-22
> 状态：已批准
> 版本：v1.0

---

## 1. 概述

### 1.1 项目背景

构建一套面向国际快递业务的 AI 销售顾问系统，实现从获客到收款的完整业务闭环。系统基于现有的 AI 平台（核心自研 + 第三方能力）和知识库，统一设计、差异化前台展示。

### 1.2 设计目标

- **一个后台**：统一的知识库、技能引擎、系统对接
- **两个前台**：被动接待模式 + 主动营销模式
- **一个闭环**：获客 → 咨询 → 报价 → 下单 → 收款 → 出库

### 1.3 业务边界

- **包含**：获客、咨询报价、下单、收款、出库前校验
- **不包含**：售后、仓库内部操作、调度优化

### 2.2 模块职责

| 模块 | 职责 | 备注 |
|------|------|------|
| 接待前台 | 接收客户主动询价，展示聊天界面 | 企业微信/网页 |

| 系统对接层 | 封装外部系统 API 调用 | 调度/CRM/出纳 |

## 3. 技能定义

### 3.1 SKILL 一览

| SKILL ID | 名称 | 功能描述 | 输入 | 输出 |
|----------|------|----------|------|------|
| SKL101 | 接待咨询 | 回答客户关于价格/仓库/流程等问题 | 客户咨询 | 解答 + 引导下单 |
| SKL102 | 智能报价 | 根据始发地/目的地/重量/仓库计算费用 | 地址+重量+仓库 | 报价单 |
| SKL103 | 下单创建 | 调用调度系统创建订单 | 客户确认信息 | 订单号 |
| SKL104 | 收款确认 | 账单生成、状态跟踪、确认到账 | 订单号 | 付款状态 |
| SKL105 | 跟单状态 | 主动推送订单状态更新 | 订单号 | 状态通知 |
| SKL106 | 催付提醒 | 未付款订单自动提醒 | 订单号 | 提醒消息 |
| SKL107 | 主动营销 | 根据线索外呼、意向确认 | 客户线索 | 转化结果 |
| SKL108 | 产品推送 | 新产品/促销信息精准推送 | 产品+客户列表 | 推送记录 |

## 4. 外部系统对接

### 4.1 调度系统 API

| 能力 | API 端点 | 用途 |
|------|----------|------|
| 查仓库 | GET /api/warehouses | 获取仓库列表 |
| 查路线 | GET /api/routes?from=&to= | 获取可用路线 |
| 查库存 | GET /api/inventory | 仓库库存查询 |
| 计算费用 | POST /api/quote | 费用试算 |
| 创建订单 | POST /api/orders | 创建运单 |
| 查询订单 | GET /api/orders/{id} | 订单状态查询 |
| 修改订单 | PUT /api/orders/{id} | 订单变更 |

### 4.2 CRM API

| 能力 | API 端点 | 用途 |
|------|----------|------|
| 客户列表 | GET /api/customers | 获取客户列表 |
| 客户详情 | GET /api/customers/{id} | 客户信息 + 分级 |
| 销售机会 | GET /api/opportunities | 重点跟进机会 |
| 更新客户 | PUT /api/customers/{id} | 更新客户标签 |

### 4.3 出纳系统 API

| 能力 | API 端点 | 用途 |
|------|----------|------|
| 创建账单 | POST /api/bills | 生成待付款账单 |
| 账单详情 | GET /api/bills/{id} | 账单状态查询 |
| 确认收款 | POST /api/bills/{id}/paid | 标记已付款 |
| 退款处理 | POST /api/bills/{id}/refund | 退款 |

### 4.4 出库校验流程

```
仓库请求出库
    │
    ▼
调度系统 ──→ 查询订单状态
    │
    ▼
出纳系统 ──→ 校验账单付款状态
    │
    ├── 已付款 ──→ 允许出库
    │
    └── 未付款 ──→ 拦截并通知 AI
                      │
                      ▼
                 SKL106 催付提醒
```

## 5. 渠道设计

### 5.1 企业微信（主渠道）

- **接待模式**：客户添加企业微信 → AI 自动通过 → 进入对话
- **消息类型**：文字/图片/链接/小程序/文件
- **主动触达**：支持每天/每周频控

### 5.2 邮件（辅助渠道）

- **适用场景**：正式报价单、账单推送、详细产品资料
- **模板管理**：可配置模板，AI 填充个性化内容

### 5.3 电话/语音（人工兜底）

- **虚拟语音**：AI 预处理后转人工，或直接转人工
- **触发条件**：客户明确要求电话、投诉、砍价等
- **人工助手**：后台接听，AI 保留上下文

### 5.4 渠道优先级

```
客户发起 → 按客户偏好渠道回复
AI 主动 → 企业微信优先 > 邮件 > 短信/电话
```

## 6. 收款闭环

### 6.1 流程图

```
┌──────────────────────────────────────────────────────────────┐
│                       收款闭环流程                            │
└──────────────────────────────────────────────────────────────┘

  订单创建                    账单生成              推送客户
  ───────                   ────────             ────────
      │                          │                   │
      ▼                          ▼                   ▼
 ┌────────┐    调用        ┌────────┐   推送    ┌────────┐
 │ 调度   │ ──创建订单──→ │ 出纳   │ ──推送──→ │ 客户   │
 │ 系统   │               │ 系统   │           │        │
 └────────┘               └────────┘           └────────┘
                                │                    │
                                │ ◄──── 等待付款 ────┘
                                │
                                ▼
                         ┌────────────┐
                         │  付款状态   │
                         │  轮询/回调  │
                         └──────┬─────┘
                                │
              ┌─────────────────┼─────────────────┐
              │                 │                 │
              ▼                 ▼                 ▼
         已付款             未付款             超时
              │                 │                 │
              ▼                 ▼                 ▼
      通知仓库放行      SKL106 催付        人工跟进
              │                 │                 │
              ▼                 ▼                 ▼
         流程完成        再次轮询          关闭订单
```

### 6.2 关键规则

| 规则 | 说明 |
|------|------|
| 出库前校验 | 仓库出库前必须确认账单已付款 |
| 催付策略 | 下单后 X 小时未付款 → 第一次提醒；Y 小时 → 第二次；Z 小时 → 人工 |
| 付款渠道 | 支持微信/支付宝/银行转账，客户自行上传凭证或系统自动对账 |
| 逾期处理 | 超时 X 天 → 自动冻结订单，通知调度系统 |

## 7. 主动营销

### 7.1 触发条件

| 类型 | 触发条件 | 优先级 |
|------|----------|--------|
| 新产品上线 | 产品发布事件 | 高 |
| 促销活动 | 促销活动配置 | 高 |
| 重点客户关怀 | 客户分级 VIP + X 天未成交 | 中 |
| 流失召回 | Y 天未下单客户 | 低 |
| 定期回访 | 货物签收后 X 天 | 中 |

### 7.2 客户分层策略

| 客户分级 | 推送策略 |
|----------|----------|
| VIP | 第一时间推送 + 人工跟进 |
| 普通 | 批量推送 + AI 自动 |
| 潜客 | 营销自动化流程 |

### 7.3 推送内容生成

- AI 根据客户历史、偏好、当前产品生成个性化话术
- 支持 A/B 测试不同话术
- 人工助手可预审推送内容

## 8. 人工助手机制

### 8.1 触发场景

| 场景 | 判断逻辑 | 处理方式 |
|------|----------|----------|
| 客户要求电话 | 识别"打电话"/"语音"意图 | 标记转人工 |
| 投诉/负面情绪 | 情感分析 < 阈值 | 标记转人工 |
| 砍价/议价 | 识别讨价还价意图 | 标记转人工 |
| 复杂业务问题 | AI 置信度 < 阈值 | 标记转人工 |
| 特定关键词 | 配置的触发词 | 标记转人工 |

### 8.2 协作流程

```
AI 对话中                    人工处理              交回 AI
─────────                  ─────────             ────────
    │                         │                    │
    ▼                         ▼                    │
识别转人工 ──────────────→ 人工接管               │
    │                         │                    │
    │                    处理完成                  │
    │                         │                    │
    │◄────────────────────────┘                    │
    │                                               │
    ▼                                               │
AI 无缝接手后续流程                                  │
    │                                               │
    └──────────────────────────────────────→ 流程继续
```

### 8.3 上下文保留

- 转人工时，AI 将完整对话上下文传递给人工助手
- 人工处理完后，AI 自动获取最新状态
- 客户无感知切换

## 11. 实施阶段

### Phase 1：接待 + 收款闭环（4-6周）

- 企业微信接入
- SKL101-104 技能开发
- 与调度系统、出纳系统对接
- 基础收款闭环

### Phase 2：主动营销（4-6周）

- SKL107-108 技能开发
- CRM 对接
- 产品推送能力
- 客户分层策略

### Phase 3：多渠道 + 人工增强（4-6周）

- 邮件渠道接入
- 电话/语音渠道
- 人工助手工作台
- 复杂场景处理

## 12. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 调度系统 API 不稳定 | 报价/下单不可用 | 本地缓存 + 降级策略 |
| 收款状态同步延迟 | 出库误放行 | 二次校验 + 人工确认 |
| AI 回答错误导致投诉 | 品牌风险 | 人工助手兜底 + 回复预审 |
| 客户骚扰/恶意下单 | 资源浪费 | 客户分级 + 频控 |

## 13. 验收标准

### 13.1 功能验收

- [ ] 客户发送"我想寄到美国" → AI 能正确报价
- [ ] 客户确认下单 → 调度系统创建订单 + 出纳系统生成账单

- [ ] 客户说"打电话给我" → 转人工助手

### 13.2 性能指标

| 指标 | 目标值 |
|------|--------|
| 首次响应时间 | < 3 秒 |
| 报价准确率 | > 95% |
| 收款确认时效 | < 1 分钟 |
| AI 转人工准确率 | > 90% |
