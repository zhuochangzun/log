## 1. 背景与目标

### 1.1 业务背景

出纳管理系统服务于企业内部财务部门，管理以下核心业务：

- **应收应付（信用额度）**：客户欠我（应收）+ 我欠供应商（应付），原系统已存在
- **资金流水**：银行/现金收支，分为有发票和无发票两类
- **发票管理**：销项发票/进项发票，与资金流水勾稽
- **固定资产折旧**：资产全生命周期管理，折旧自动计提
- **财务报表**：资产负债表、利润表、现金流量表
- **纳税申报**：全税种税务申报表

### 1.2 系统目标

| 目标 | 描述 |
|------|------|
| 三流合一 | 信用额度流、发票流、资金流水统一管理 |
| 自动对账 | 发票与资金流水自动匹配核销 |
| 多公司多账号 | 支持集团化多公司、多银行账户管理 |
| 多币种支持 | 额度、流水、发票支持多币种 |
| 报表审批 | 财务报表、纳税申报需审批后上报 |
| 固定资产联动 | 折旧计算结果自动生成凭证 |

## 2. 组织架构

### 2.1 多公司多账号

```
公司 (COMPANY)
    │
    └── 账号 (ACCOUNT) ← 本位币
            │
            ├── 信用额度 (CREDIT_LIMIT)
            │       ├── 应收 (RECEIVABLE) ← 客户欠我
            │       ├── 应付 (PAYABLE) ← 我欠供应商
            │       └── 共享额度池 (CREDIT_POOL) ← 关联后合并
            │
            ├── 供应商-客户关联 (CREDIT_RELATION)
            │       └── 供应商 ←→ 客户（可多对多）
            │
            ├── 资金流水 (FUND_TRANSACTION)
            ├── 固定资产 (FIXED_ASSET)
            ├── 发票 (INVOICE)
            └── 报表 (FINANCIAL_STATEMENT/TAX_REPORT)
```

**公司 (COMPANY)**：独立法人实体，独立账务
**账号 (ACCOUNT)**：公司下的银行账户/现金账户，支持多币种

## 3. 核心数据模型

### 3.1 资金流三分类

| 类型 | 说明 | 勾稽关系 |
|------|------|---------|
| 信用额度 | 应收应付（原系统同步） | 额度 → 发票 → 收付款 |
| 有发票流水 | 业务收支已开票 | 与发票一一对应 |
| 无发票流水 | 日常杂项收支 | 独立记录 |

### 3.2 核心表结构

| 表名 | 说明 |
|------|------|
| `COMPANY` | 公司主数据（名称、税号、法人等） |
| `ACCOUNT` | 账号主数据（公司下银行/现金账户，本位币） |
| `CREDIT_LIMIT` | 信用额度台账（从原系统同步） |
| `FUND_TRANSACTION` | 资金流水（有/无发票统一管理） |
| `INVOICE` | 发票主表 |
| `INVOICE_LINE` | 发票明细 |
| `RECONCILIATION` | 对账记录（自动完成） |
| `JOURNAL_ENTRY` | 凭证/日记账 |
| `FIXED_ASSET` | 固定资产主数据 |
| `FIXED_ASSET_DEPRECIATION` | 固定资产折旧记录 |
| `APPROVAL_TASK` | 审批任务 |
| `APPROVAL_RECORD` | 审批记录 |
| `FINANCIAL_STATEMENT` | 财务报表 |
| `TAX_REPORT` | 纳税申报表 |
| `OCR_INVOICE_IMAGE` | OCR发票图片原始数据 |
| `CREDIT_RELATION` | 供应商-客户额度关联关系 |
| `CREDIT_POOL` | 共享额度池 |

## 5. 状态机设计

### 5.1 发票状态

```
发票登记 → 待确认 → 已确认 → 部分核销 → 已核销
                          ↓
                     异常作废
```

### 5.2 对账状态（自动完成）

```
自动匹配 → 已对账
    ↓
未匹配项 → 人工处理（可选）
```

### 5.3 报表状态

```
报表生成 → 待审批 → 已审批 → 已上报
                ↓
            审批驳回 → 重新生成
```

### 5.4 固定资产状态

```
购置 → 使用中 → 折旧计提 → 折旧完成
              ↓
           资产处置 → 已清理
```

### 5.5 OCR 发票识别

```
扫描上传 → OCR识别 → 信息提取 → 人工确认 → 发票登记
```

## 6. 关键设计决策

1. **多公司多账号** — 公司 → 账号，账号独立本位币
2. **信用额度增量同步** — 只同步变化量，不全量覆盖
3. **资金流水二分法** — 有发票 vs 无发票（各自独立对账）
4. **应收应付统一** — 信用额度同时记录"欠人"和"被人欠"
5. **多币种折算** — 按账号本位币实时折算
6. **对账自动完成** — 系统自动匹配，无需人工确认
7. **报表审批流程** — 财务报表、纳税申报需审批
8. **发票纳税联动** — 发票确认后自动计算应纳税额
9. **会计期间报表** — 支持月报、季报、年报
10. **固定资产折旧联动** — 折旧计算结果自动生成凭证
