# 知识分类体系设计

## 1. 三层知识架构

```
平台层（全局共享）
    │
    ▼
公司层（租户隔离）
    │
    ├── 订单层（订单数据）
    │
    └── 用户层（个人数据）
```

### 1.1 层级说明

| 层级 | 数据归属 | 可见范围 | 典型用户 |
|------|---------|---------|---------|
| 平台层 | 睿云平台 | 所有公司/员工/用户 | 货代公司员工、货代客户 |
| 公司层 | 货代公司 | 仅该公司内 | 货代公司员工、货代客户 |
| 订单层 | 货代公司 | 按订单阶段隔离 | 货代公司员工 |
| 用户层 | 个人用户 | 仅本人（可授权） | 货代客户 |

---

## 2. 平台层分类

睿云平台管理的行业基础知识，开放给所有用户。

### 2.1 分类定义

| 分类 ID | 分类名称 | 说明 | 示例 |
|---------|---------|------|------|
| PLT001 | 快递公司 | 市场主流快递公司信息 | FedEx/DHL/UPS/顺丰... |
| PLT002 | 电商平台 | 主流电商平台物流政策 | Amazon/eBay/Shopify... |
| PLT003 | 海关政策 | 各国家/地区进出口规定 | 中国出口、欧美进口... |
| PLT004 | 物流方案 | 物流方案百科 | FBA头程、海外仓、直邮... |
| PLT005 | 行业术语 | 物流专业名词解释 | CIF/FOB/ENS/BL... |
| PLT006 | 竞品动态 | 行业新闻、竞品分析 | 某公司新服务、涨价通知... |

### 2.2 分类管理

- 分类由睿云平台管理员创建/修改/删除
- 支持启用/禁用分类（禁用后不显示但不删除数据）
- 支持分类排序（影响前端显示顺序）

---

## 3. 公司层分类

货代公司管理的客户数据，按租户隔离。

### 3.1 分类定义

| 分类 ID | 分类名称 | 说明 | 示例 |
|---------|---------|------|------|
| CO001 | 公司资料 | 资质、协议、合作条款 | 营业执照、合作协议... |
| CO002 | 产品信息 | 航线、时效、价格表 | 美国专线-空派、14天... |
| CO003 | 客户信息 | 客户档案、联系人 | 客户档案、委托记录... |
| CO004 | SOP流程 | 内部操作标准流程 | 接单SOP、报关SOP... |
| CO005 | 报价规则 | 折扣政策、账期规则 | VIP折扣、月结30天... |
| CO006 | 常见问题 | 客户高频问题回复 | 查不到运单怎么办... |

### 3.2 分类管理

- 分类由公司管理员创建/修改/删除
- 支持启用/禁用分类
- 支持分类排序

---

## 4. 订单层分类

按订单处理阶段划分，同属公司层数据。

### 4.1 分类定义

| 分类 ID | 分类名称 | 说明 |
|---------|---------|------|
| ORD001 | 待处理 | 新下单未确认 |
| ORD002 | 执行中 | 已确认处理中 |
| ORD003 | 已完成 | 正常完结 |
| ORD004 | 异常 | 问题件、退件等 |

### 4.2 订单与公司层关系

- 订单数据归属公司层
- 订单沿用公司层的 SOP、常见问题等分类进行关联
- 订单有独立的处理阶段状态

---

## 5. 用户层分类

个人用户的数据，支持公司授权共享。

### 5.1 分类定义

| 分类 ID | 分类名称 | 说明 | 共享设置 |
|---------|---------|------|---------|
| USR001 | 个人话术 | 常用回复模板 | 可授权给公司 |
| USR002 | 常用模板 | 邮件/消息模板 | 可授权给公司 |
| USR003 | 学习笔记 | 个人学习记录 | 不可共享 |
| USR004 | 收藏内容 | 收藏的知识条目 | 不可共享 |

### 5.2 授权机制

- 用户可将自己的话术/模板授权给公司
- 被授权的内容对公司内其他用户可见
- 授权可随时撤回

---

## 6. 分类管理 API

### 6.1 平台层（管理员）

```
POST   /admin/categories              创建分类
GET    /admin/categories              列表
PUT    /admin/categories/{id}        更新
DELETE /admin/categories/{id}        删除
PATCH  /admin/categories/{id}/toggle 启用/禁用
PATCH  /admin/categories/sort        排序
```

### 6.2 公司层（公司管理员）

```
POST   /company/categories            创建分类
GET    /company/categories            列表
PUT    /company/categories/{id}      更新
DELETE /company/categories/{id}      删除
PATCH  /company/categories/sort      排序
```

### 6.3 用户层（普通用户）

```
POST   /user/categories               创建分类（仅用户层）
GET    /user/categories              列表
PUT    /user/categories/{id}         更新
DELETE /user/categories/{id}         删除
```

## 7. 分类数据结构

### 7.1 分类实体

```json
{
  "id": "PLT001",
  "name": "快递公司",
  "layer": "platform",
  "parent_id": null,
  "description": "市场主流快递公司信息",
  "sort_order": 1,
  "status": "active",
  "created_at": "2026-03-28T10:00:00Z",
  "updated_at": "2026-03-28T10:00:00Z"
}
```

### 7.2 层级枚举

| 值 | 说明 |
|----|------|
| `platform` | 平台层 |
| `company` | 公司层 |
| `order` | 订单层 |
| `user` | 用户层 |
